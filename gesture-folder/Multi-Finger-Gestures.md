الفصل: الإيماءات متعددة الأصابع (Multi-Finger Gestures) في مكتبة Gesture Kit
الجزء الأول: الأسس النظرية والتحليل الرياضي لإيماءات Pinch و Rotation
________________________________________
1. مقدمة عامة
تُعد الإيماءات متعددة الأصابع (Multi-Finger Gestures) من أكثر أنماط التفاعل تطورًا في الأنظمة الحديثة، لأنها تعتمد على استخدام إصبعين أو أكثر في الوقت نفسه لتنفيذ أوامر معقدة لا يمكن تحقيقها بسهولة عبر اللمس الأحادي (Single-Touch).
يُستخدم هذا النوع من الإيماءات بشكل واسع في:
•	تطبيقات الخرائط (تكبير وتصغير)
•	عرض الصور
•	التصميم الهندسي
•	الواقع المعزز
•	النماذج ثلاثية الأبعاد
على عكس الإيماءات التقليدية التي تعتمد على نقطة لمس واحدة، فإن Multi-Finger Gestures تعتمد على العلاقة الرياضية بين عدة نقاط لمس، مثل:
•	المسافة بين الإصبعين
•	الزاوية بينهما
•	اتجاه الحركة
•	المركز الهندسي (Midpoint)
________________________________________
2. أهمية Multi-Finger Gestures في التطبيقات الحديثة
تكمن أهمية هذه الإيماءات في أنها تمنح المستخدم قدرة أكبر على:
•	التحكم في الحجم (Scaling)
•	الدوران (Rotation)
•	التنقل الدقيق
•	التفاعل الطبيعي مع العناصر الرسومية
أمثلة عملية:
•	Pinch Out → تكبير صورة
•	Pinch In → تصغير خريطة
•	Rotation Clockwise → تدوير عنصر
•	Combined Pinch + Rotation → تعديل نموذج ثلاثي الأبعاد
________________________________________
3. الفرق بين Multi-Finger و Single-Touch Gestures
العنصر	Single-Touch	Multi-Finger
عدد نقاط الإدخال	إصبع واحد	إصبعان أو أكثر
التعقيد	بسيط	مرتفع
الاستخدام	Tap / Drag	Zoom / Rotate
البيانات المطلوبة	Position فقط	Position + Distance + Angle

4. الخلفية النظرية (Theoretical Background)
لفهم Multi-Finger Gestures، يجب تحليلها رياضيًا وهندسيًا.
________________________________________
4.1 نظام الإحداثيات اللمسية (Touch Coordinate System)
كل إصبع على الشاشة يمثل نقطة:
Finger A = (x1, y1)
Finger B = (x2, y2)
وبالتالي يصبح النظام قائمًا على تحليل العلاقة بين نقطتين.
________________________________________
4.2 حساب المسافة بين الإصبعين (Distance Formula)
المسافة بين الإصبعين هي الأساس في Pinch Gesture:
d = √((x2 - x1)² + (y2 - y1)²)
________________________________________
التفسير:
•	إذا زادت المسافة → Zoom In
•	إذا نقصت المسافة → Zoom Out
________________________________________
مثال:
إذا كانت:
•	المسافة السابقة = 100
•	المسافة الحالية = 150
فهذا يعني:
Scale Up بنسبة 1.5
________________________________________
4.3 عامل التكبير (Scale Factor)
scale = currentDistance / initialDistance
________________________________________
حالات:
•	scale > 1 → تكبير
•	scale < 1 → تصغير
•	scale = 1 → لا تغيير
________________________________________
4.4 النقطة المركزية (Midpoint)
تمثل مركز الإيماءة:
midX = (x1 + x2) / 2
midY = (y1 + y2) / 2
________________________________________
أهمية Midpoint:
•	تحديد مركز التكبير
•	تحسين تجربة المستخدم
•	الحفاظ على استقرار العنصر أثناء التكبير
________________________________________
5. الأساس النظري لإيماءة Rotation
5.1 مفهوم الدوران
تعتمد Rotation Gesture على التغير في الزاوية بين الإصبعين.
________________________________________
5.2 حساب الزاوية
θ = atan2(y2 - y1, x2 - x1)
________________________________________
لماذا atan2؟
لأنها:
•	تعطي الزاوية الصحيحة
•	تحدد الاتجاه
•	تعمل في جميع الأرباع
________________________________________
التغير الزاوي:
rotation = currentAngle - initialAngle
________________________________________
الحالات:
•	rotation > 0 → دوران مع عقارب الساعة
•	rotation < 0 → دوران عكس عقارب الساعة
________________________________________
6. التحليل الفيزيائي للحركة متعددة الأصابع
6.1 الحركة النسبية (Relative Motion)
لا يتم تحليل كل إصبع منفردًا، بل يتم تحليل:
•	التغير في المسافة
•	التغير في الزاوية
________________________________________
6.2 الحركة المتزامنة
قد يحدث:
•	Pinch فقط
•	Rotation فقط
•	Pinch + Rotation معًا
وهذا يتطلب نظامًا قادرًا على فصل الأنماط.

الجزء الثاني: آلية العمل الداخلية (Internal Working Mechanism)
________________________________________
1. مقدمة
بعد استعراض الأساس النظري والرياضي لإيماءات Multi-Finger، تأتي المرحلة الأهم وهي فهم كيف يعمل النظام داخليًا لتحويل حركات الإصبعين إلى أوامر فعلية مثل التكبير (Pinch) أو التدوير (Rotation).
تكمن صعوبة هذا النوع من الإيماءات في أن النظام لا يتعامل مع نقطة لمس واحدة، بل مع عدة نقاط متزامنة يجب تتبعها وتحليل العلاقة بينها لحظيًا.
________________________________________
2. نظرة عامة على تدفق الأحداث (Event Flow)
يمر النظام بعدة مراحل أساسية:
Touch Input → Pointer Tracking → Coordinate Extraction → Distance/Angle Calculation → Threshold Analysis → Gesture Recognition → Animation Output
________________________________________
3. مرحلة التقاط الإدخال (Touch Input Capture)
3.1 تعدد المؤشرات (Multiple Pointers)
عند استخدام إصبعين، يقوم النظام بتسجيل:
•	Pointer 1
•	Pointer 2
كل Pointer يحتوي على:
•	x
•	y
•	id
•	state
________________________________________
3.2 أهمية Pointer IDs
يجب تتبع كل إصبع بشكل مستقل، لأن تبديل الترتيب قد يؤدي إلى:
•	أخطاء زاوية
•	انعكاس الاتجاه
•	نتائج غير مستقرة
________________________________________
4. تتبع النقاط (Pointer Tracking)
الهدف:
الحفاظ على هوية كل إصبع أثناء الحركة.
________________________________________
مثال منطقي:
إذا بدأ المستخدم بـ:
Finger A → Pointer 1
Finger B → Pointer 2
فيجب الحفاظ على نفس الترتيب حتى النهاية.
________________________________________
5. استخراج الإحداثيات (Coordinate Extraction)
const x1 = event.touches[0].x;
const y1 = event.touches[0].y;

const x2 = event.touches[1].x;
const y2 = event.touches[1].y;
________________________________________
التحليل:
•	يتم أخذ موضع كل إصبع
•	تُستخدم القيم للحسابات الهندسية
________________________________________
6. حساب المسافة (Pinch Engine)
const distance = Math.sqrt(
  Math.pow(x2 - x1, 2) + Math.pow(y2 - y1, 2)
);
________________________________________
الاستخدام:
•	مقارنة المسافة الحالية بالمسافة الابتدائية
•	استخراج Scale Factor
________________________________________
مثال:
Initial = 120
Current = 180
Scale = 180 / 120 = 1.5
________________________________________
7. حساب الزاوية (Rotation Engine)
const angle = Math.atan2(y2 - y1, x2 - x1);
________________________________________
الاستخدام:
•	تحديد زاوية الإصبعين
•	مقارنة الزاوية الحالية بالزاوية الأصلية
________________________________________
مثال:
Initial = 20°
Current = 50°
Rotation = 30°
________________________________________
8. تحليل التغير (Delta Analysis)
8.1 Delta Distance
Δd = currentDistance - initialDistance
________________________________________
8.2 Delta Angle
Δθ = currentAngle - initialAngle
________________________________________
الهدف:
تمييز:
•	تكبير
•	تصغير
•	تدوير

9. Threshold Conditions
9.1 Threshold Scale
إذا كان:
|Δd| < MinDistanceThreshold

________________________________________
9.2 Threshold Rotation
إذا كان:

10. Simultaneous Recognition

6. الكود البرمجي
import { Gesture } from 'react-native-gesture-handler';
import { useSharedValue } from 'react-native-reanimated';

export const usePinch = () => {
  const scale = useSharedValue(1);

  const pinchGesture = Gesture.Pinch()
    .onUpdate((event) => {
      scale.value = event.scale;
    });

  return { scale, pinchGesture };
};
