الفصل: إيماءات السحب والحركة (Drag, Pan, Swipe, Fling Gestures) في مكتبة Gesture Kit
الجزء الأول: الأسس النظرية والتحليل المفاهيمي
________________________________________
1. مقدمة
تُعد إيماءات السحب والحركة (Drag & Motion Gestures) من أكثر فئات الإيماءات استخدامًا في التطبيقات التفاعلية الحديثة، لأنها تمثل الأساس المباشر للتفاعل الحركي بين المستخدم والعناصر الرقمية على الشاشة.
فعندما يقوم المستخدم بتحريك عنصر، سحب بطاقة، تمرير قائمة، أو قذف عنصر بسرعة، فإن النظام يعتمد على مجموعة من الإيماءات المرتبطة بالحركة، لكل منها خصائص منطقية وفيزيائية مختلفة.
تشمل هذه الفئة أربع إيماءات رئيسية:
1.	Drag Gesture
2.	Pan Gesture
3.	Swipe Gesture
4.	Fling Gesture
ورغم التشابه الظاهري بينها، إلا أن كل نوع يعتمد على معايير مختلفة مثل:
•	المسافة (Distance)
•	السرعة (Velocity)
•	الاتجاه (Direction)
•	الزمن (Duration)
•	الزخم (Momentum)
________________________________________
2. أهمية Drag & Motion Gestures في التطبيقات الحديثة
تستخدم هذه الإيماءات في:
•	سحب العناصر داخل الواجهات
•	التنقل بين الصفحات
•	تمرير القوائم
•	حذف العناصر بالسحب
•	الألعاب
•	أنظمة البطاقات
•	الأنيميشن التفاعلي
________________________________________
أمثلة:
•	Drag → نقل عنصر
•	Pan → تحريك مساحة كبيرة
•	Swipe → انتقال سريع
•	Fling → قذف عنصر بسرعة مع استمرار الحركة
________________________________________
3. الفرق بين أنواع الإيماءات الحركية
الإيماءة	تعتمد على	الهدف
Drag	Position Change	تحريك عنصر
Pan	Continuous Translation	تحريك مساحة
Swipe	Direction + Speed	أمر سريع
Fling	High Velocity + Momentum	قذف

4. الخلفية النظرية (Theoretical Background)
لفهم هذه الإيماءات، يجب فهم أربعة مفاهيم أساسية:
________________________________________
4.1 الإزاحة (Translation)
تمثل مقدار انتقال العنصر:
Δx = currentX - startX
Δy = currentY - startY
________________________________________
الاستخدام:
•	Drag
•	Pan
________________________________________
4.2 الاتجاه (Direction)
يحدد مسار الحركة:
•	Left
•	Right
•	Up
•	Down
________________________________________
التحليل:
إذا:
|Δx| > |Δy|
→ حركة أفقية
إذا:
|Δy| > |Δx|
→ حركة عمودية
________________________________________
4.3 السرعة (Velocity)
السرعة عنصر أساسي خاصة في Swipe وFling:
v = distance / time
________________________________________
الاستخدام:
•	سرعة منخفضة → Drag
•	سرعة متوسطة → Swipe
•	سرعة عالية → Fling
________________________________________
4.4 الزخم (Momentum)
يمثل استمرار الحركة بعد إفلات الإصبع.
________________________________________
مثال:
عند قذف بطاقة بسرعة، تستمر بالحركة بعد انتهاء اللمس.

الفصل: إيماءات السحب والحركة (Drag, Pan, Swipe, Fling Gestures) في مكتبة Gesture Kit
الجزء الثاني: آلية العمل الداخلية (Internal Working Mechanism)
________________________________________
1. مقدمة
بعد استعراض الأسس النظرية لإيماءات Drag & Motion، تأتي المرحلة الأهم وهي فهم كيف يلتقط النظام الحركة، يعالجها، ويفسرها لاكتشاف نوع الإيماءة بدقة.
رغم أن المستخدم يرى مجرد “سحب” أو “تمرير”، إلا أن النظام داخليًا ينفذ سلسلة معقدة من العمليات الحسابية والتحليلية لتحويل بيانات اللمس الخام إلى أوامر مثل:
•	Drag
•	Pan
•	Swipe
•	Fling
________________________________________
2. التدفق الداخلي للنظام (Event Flow)
يمر النظام بالمراحل التالية:
Touch Start → Pointer Tracking → Position Sampling → Translation Analysis → Velocity Calculation → Direction Detection → Threshold Evaluation → Gesture Classification → Animation Output
________________________________________
3. مرحلة التقاط اللمس (Touch Capture Layer)
3.1 Touch Start
عند ملامسة الشاشة:
•	تسجيل نقطة البداية
•	حفظ الوقت الابتدائي
startX = event.x;
startY = event.y;
startTime = Date.now();
________________________________________
الوظيفة:
هذه القيم تمثل المرجع الأساسي لجميع الحسابات اللاحقة.
________________________________________
4. مرحلة التتبع (Pointer Tracking)
الهدف:
تحديث موضع الإصبع بشكل مستمر أثناء الحركة.
currentX = event.x;
currentY = event.y;
________________________________________
النتيجة:
يصبح لدينا:
•	Start Position
•	Current Position
________________________________________
5. حساب الإزاحة (Translation Engine)
translateX = currentX - startX;
translateY = currentY - startY;
________________________________________
التحليل:
•	Drag يعتمد مباشرة على هذه القيم
•	Pan يستخدمها بشكل مستمر
________________________________________
6. حساب المسافة الكلية (Distance Calculation)
distance = Math.sqrt(
  translateX * translateX +
  translateY * translateY
);
________________________________________
الاستخدام:
•	هل الحركة كافية؟
•	هل تجاوزت الحد الأدنى؟
________________________________________
7. حساب الزمن
duration = Date.now() - startTime;
________________________________________
الاستخدام:
•	مهم جدًا لـ Swipe وFling
________________________________________
8. حساب السرعة (Velocity Engine)
velocity = distance / duration;
________________________________________
التحليل:
سرعة منخفضة:
Drag
سرعة متوسطة:
Swipe
سرعة عالية:
Fling
________________________________________
9. تحليل الاتجاه (Direction Detection)
if (Math.abs(translateX) > Math.abs(translateY)) {
  direction = translateX > 0 ? "RIGHT" : "LEFT";
} else {
  direction = translateY > 0 ? "DOWN" : "UP";
}
________________________________________
الوظيفة:
تحديد نوع الحركة:
•	أفقي
•	عمودي
________________________________________
10. Threshold Logic
________________________________________
10.1 Drag Threshold
يتطلب حركة بسيطة
________________________________________
10.2 Swipe Threshold
يتطلب:
•	Distance > X
•	Velocity > Y
________________________________________
10.3 Fling Threshold
يتطلب:
•	Velocity مرتفعة جدًا
________________________________________
مثال:

  gesture = "FLING";
}

المنطق:

  return "FLING";
}
if (velocity > swipeThreshold) {
  return "SWIPE";
}
return "DRAG";

5. الكود
import { Gesture } from 'react-native-gesture-handler';
import { useSharedValue } from 'react-native-reanimated';

export const useDrag = () => {
  const translateX = useSharedValue(0);
  const translateY = useSharedValue(0);

  const dragGesture = Gesture.Pan()
    .onUpdate((event) => {
      translateX.value = event.translationX;
      translateY.value = event.translationY;
    });

  return { translateX, translateY, dragGesture };
};

export const useSwipe = (onSwipe) => {
  const swipeGesture = Gesture.Pan()
    .onEnd((event) => {
      if (Math.abs(event.translationX) > 50 &&

        onSwipe(event.translationX > 0 ? "RIGHT" : "LEFT");

    });

  return { swipeGesture };
};

export const useDrag = () => {
  const x = useSharedValue(0);
  const y = useSharedValue(0);

  const gesture = Gesture.Pan()
    .onUpdate((e) => {
      x.value = e.translationX;
      y.value = e.translationY;
    });

  return { x, y, gesture };
};
